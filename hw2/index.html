Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

What is de Casteljau’s algorithm:
De casteljau’s algorithm evaluates a Bezier curve at a given parameter t, where t is between 0 and 1, by repeatedly interpolating adjacent points together. Starting from n original points, the algorithm interpolates the first and second point, second and third point, and so on until the last point. This process reduces the total number of points by 1. The algorithm then repeats on the new set of points, until we arrive at a final point. This point is the point of the Bezier curve at a specific parameter t. Note that t is a parameter that represents the weight of one point over another (i.e. a t value of 0.5 would be equivalent to interpolating two points by finding their midpoint, and a t value of 0.1 would mean finding a point that divides the line segment between the two points into a 1:9 ratio for instance).

How I implemented de Casteljau’s algorithm:
In BezierCurve::evaluateStep, I compute just one subdivision level. For each adjacent pair of points points[i] and points[i + 1], I compute the interpolated point (1 - t) * points[i] + t * points[i + 1] and add it to a vector. This function is repeatedly called when I press “E”, which makes it ultimately produce a single evaluated point on the Bezier curve.

Part 2: Bezier surfaces with separable 1D de Casteljau

How de Casteljau algorithm extends to Bezier surfaces:
The de Casteljau algorithm can be extended from Bezier curves to Bezier surfaces by applying the same 1D interpolation process two times, in two different directions. A Bezier surface is defined by an n by n grid of control points and two parameters u and v, each ranging from 0 to 1. Each row of control points is treated as a Bezier curve parameterized by u. Using the same recursive interpolation step from Part 1, de Casteljau’s algorithm is applied to each row to compute a single intermediate point for that row. This results in n points that lie along a column of the surface. Next, these n intermediate points are treated as the control points of another Bezier curve, this time parameterized by v. Applying de Casteljau’s algorithm again to these points produces a final single point, which lies on the Bezier surface at the parameter (u,v).

How I implemented it:
I first implemented BezierPatch::evaluateStep, which performs one layer of linear interpolation between adjacent control points in 3D. Then BezierPatch::evaluate1D repeatedly calls evaluateStep until only one point remains, fully evaluating a Bezier curve at a given parameter. Finally, in BezierPatch::evaluate, I evaluate each row of the control point grid at parameter u to obtain intermediate points, and then evaluate those points at parameter v to compute the final point on the Bezier surface.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

How I implemented area weighted vertex normals:
Area-weighted vertex normals are computed by averaging the normals of all faces incident to a vertex, where each face normal is weighted by the area of the face. Using the half-edge data structure, I start from a halfedge associated with the vertex and iterate around all neighboring faces by repeatedly following the twin() and next() pointers. This allows me to visit every triangle that shares the vertex. At each triangle, I compute two edge vectors and take their cross product. The cross product produces a vector perpendicular to the triangle’s surface, and its magnitude is proportional to the area of the triangle. I add these vectors together for all incident faces, then normalize the result to obtain a unit-length vertex normal.

Part 4: Edge flip

How I implemented the edge flip operation:
An edge flip is an operation that replaces the shared edge between two adjacent triangles with the other diagonal of the quadrilateral formed by the four vertices of those triangles. If two triangles share edge (b,c), flipping that edge replaces it with edge (a,d), producing two new triangles while keeping the same four vertices. To implement this in the half-edge data structure, I first retrieved the halfedge of the selected edge and its twin, along with the other four halfedges that form the two neighboring triangles. If either adjacent face is a boundary face, I return since boundary edges should not be flipped. Otherwise, I reassign the connectivity of the six halfedges involved in the operation and update their next, vertex, face, and edge pointers so that the two faces are rewired correctly after the flip. I also update the halfedge pointers for the affected vertices, faces, and the flipped edge so that each element still points to a valid halfedge.

Debugging:
The most helpful trick was drawing the two triangles and labeling all six halfedges before and after the flip. This made it easier to reason about which pointers needed to change. I also made sure to explicitly set all pointers for the affected halfedges instead of only the ones I thought were changing, since missing even a single pointer can lead to broken connectivity or holes.

Part 5: Edge split

How I implemented the edge split operation:
An edge split takes two triangles sharing an interior edge (b,c), inserts a new vertex m at the midpoint of that edge, and re-triangulates the local neighborhood into four triangles by connecting m to the two opposite vertices a and d. In my implementation, I first grabbed the halfedge of the target edge and its twin, along with the other four halfedges that complete the two adjacent triangles. I ignored boundary edges by returning immediately if either adjacent face is boundary. Then, I created the new vertex m and set its position to the midpoint of the two endpoints of the original edge. Next I allocated the new edges, faces, and halfedges and rewired all local pointers. Finally, I set m->halfedge() to one of the halfedges on the original edge object after the split as required.

Debugging:
Similar to Part 4, drawing and labeling every local halfedge before and after the split was the most helpful approach. I also repeatedly split edges, flipped nearby edges, and alternated split/flip operations to ensure the mesh stayed valid without cracks or crashes.

Part 6: Loop subdivision for mesh upsampling

How I implemented loop subdivision:
Loop subdivision upsamples a triangle mesh by subdividing each triangle into four smaller triangles and updating vertex positions using a weighted averaging rule. I first computed updated positions for all original vertices and stored them in Vertex::newPosition. I also computed positions for new edge vertices in Edge::newPosition. After that, I split every original edge and flipped any new edge connecting an old and new vertex to produce the correct subdivision pattern. Finally, I copied all newPosition values into final vertex positions.

Debugging:
The main nuance was storing a list of original edges before splitting, which prevents newly created edges from being split again in the same iteration. I also repeatedly applied subdivision and checked for holes, broken connectivity, and incorrect flips.

How meshes behave after subdivision:
After applying loop subdivision several times, the mesh becomes smoother and the number of triangles increases significantly. Sharp edges and corners gradually become rounded because vertex positions move toward weighted averages of neighboring vertices. One way to reduce this effect is by pre-splitting edges near sharp features before subdivision.

How pre-processing helps alleviate asymmetries:
When repeatedly applying loop subdivision to a cube, it becomes smoother and more rounded, but can also become slightly asymmetric. This happens because each square face is triangulated by a diagonal, and that orientation introduces directional bias that accumulates over repeated subdivision. I pre-processed by flipping edges so triangulation across faces is more symmetric. This distributes updates more evenly and yields a visibly more symmetric result after subdivision.
